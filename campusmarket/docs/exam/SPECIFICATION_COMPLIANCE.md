## Submission package

- Printed report source and export following the mandated chapter structure.
- Presentation with no more than 20 slides.
- Seven-minute demonstration script and captured evidence.
- ZIP containing source, configuration, documentation, reports, and evidence.
- GitHub repository and public deployment links.

## Production constraints

- Application workloads run on the Azure VPS under K3s, never on the assessor's
  workstation.
- Jenkins remains a host service on the same VPS and deploys to the local K3s
  API through `/etc/rancher/k3s/k3s.yaml`.
- Secrets are Jenkins credentials or Kubernetes Secrets created from the
  protected VPS environment file; no secret values enter Git.
- Prometheus, Grafana, and SonarQube are administrative surfaces and are not
  exposed through public Azure rules.
- Every cutover requires a database backup, rollout health checks, public smoke
  tests, and a documented rollback command.
